Quickstart example of basic procedure

If anything feels confusing, it may clear up from the practical example below. It may also help to look at some pictures of Enigma machines to maybe get an understanding of what I'm trying to show:

The emulator consists of four parts, from top to bottom:

  1. Controls for internal settings, and rotor setting displays
  2. Lampboard which lights up each processed letter
  3. Keyboard which is used to input letters
  4. Plugboard
  5. Notepad, which updates automatically when letters are processed

The four gray bars beside the yellow letter windows are the rotors. They can be shifted up or down by clicking the upper or lower part.

The option "Keep light on" can be used to keep the lampboard shining when a key is released. This makes it a bit easier to use, and is necessary for use with touch screens.

Basic usage

For two operators to communicate, they both need to have the same settings configured. These settings include:

  • Reflector selection (Umkehrwalze)
  • Rotor types and order (Walzenlage)
  • Ring setting (Ringstellung)
  • Plugboard connections (Steckerverbindungen)

To configure reflector, rotors, and ring settings, use the controls on the upper part. To connect plugs on the plugboard, click two letters on the board in succession. To reset a plugboard connection, simply click one of its plugs.

Practical example

This is a simplified example. The correct procedure is covered below.

Say you want to encrypt the sentence "Cryptography is fun". First you choose a setting, for example:

  • Reflector: C
    • Choose B or C. The M4 Enigma uses a thin rotor here.
  • Rotors: β IV VII III
    • The first rotor is special for the M4, choose from β and γ. Then choose three rotors between I and VIII. An actual Enigma machine only comes with one rotor of each.
  • Ring setting: 01 01 01 16
    • This represents an internal rotation for each rotor, choose between 01 and 26.
  • Plugboard connections: DO BX QM EZ AP FL IR UH
    • These are letter pairs connected with wires between each corresponding plug. A letter cannot be connected to itself, and a pair can only be connected to each other.
  • Key: MCQD
    • Choose four random letters, this is the encryption key. Rotate the rotors so that these letters are shown in the four windows.

Then you prepare the plaintext by removing the spaces, and group the letters in fours for clarity:

CRYP TOGR APHY ISFU N

Then you type each letter on the Enigma keyboard, holding down the key and noting which lamp lights up. In the emulator, notes are taken automatically on the notepad. You should get the result:

MECL CAQG KVVH DAPL T

You're done! The message can now be sent to another operator, provided they know the settings and the key. How they would know this is covered below, along with the actual encryption procedure which is more complex.

To decrypt it, set the machine to the same settings as when you encrypted the message, and simply type the ciphertext on the Enigma keyboard. You should get the plaintext as result.

The machine settings for this message was:

  • Reflector: B
  • Rotors: β II IV I
  • Ring setting: 01 01 01 22
  • Plugboard connections: AT BL DF GJ HM NW OP QY RZ VX
  • Key: VJNA

Cleaned up, suspected operator typos marked bold:

From "Looks" radio-telegram 1132/19. Contents: Under attack, submerged, depth charges. Last location 08:30 hours sea square AJ9863, course 220, 8 knots, following. Pressure: 14 millibar falling, wind NNE 4 m/s, visibility 10.

Preparation of the Plaintext

Note: This is based on the translated 1940 Kriegsmarine version of the Enigma general procedure.

The M4 Enigma machine's keyboard only contains letters. No digits, punctuation marks, or spacebar. This means that these characters need to be replaced with letters, according to the following procedure.

The plaintext should be written so that it is clear and easy to interpret, and avoid any confusion.

Where it is possible without risk of confusion, variations should be used to make cryptanalysis harder. Example: "Minensuchboot" can be written as MINENSUCHBOOT, MINBOOT, or MMMBOOT.

Spaces

Spaces are simply omitted. Note: examples below will be written with spaces anyway, for clarity.

Stress Marks

Special words are noted by stress marks (J), for example location "Cap Gris Nez" is written J CAP GRIS NEZ J. Especially important words are written twice along with the stress marks: J CAP GRIS NEZ J J CAP GRIS NEZ J.

Abbreviations

To avoid confusionand errors in abbreviations, distinguishing letters are given thrice: "B.d.U."" is written B D UUU (or BBB DDD UUU)

Periods are left out.

Numbers

Numbers are typed out, most often with each digit in succession. Some multiple digit numbers can be written as for example: ZEHN, ELF, ZWOELF, ZWANZIG, DREISSIG... and HUNDERT, TAUSEND...

Series of numbers which would be separated by commas, are separated with Y: "M 4, 7, 11 und 25" is written MMM VIER Y SIEBEN Y ELF UND ZWO FUNF

If the original text contains number series without commas, the commas must be inserted to avoid confusion, for example "3 1 6" should be written DREI Y EINS Y SEQS.

List of some common text aids

J-----J Stress marks
X Period
Y Comma
UD Question mark
XX Colon
KK Parenthesis
YY Hyphen, dash, or slash

Message length

Messages should be made to vary in length by adding letters and words at the end. After the last word of the plaintext, three or four identical bigrams (letter pairs) are to be added. After these a common neutral word should be added.

Example: ...ABGESUCHT KB KB KB GARDEROBENSTANDER

Words which are contrary to "good breeding(sic)" and good order, personal communications, and the like, are forbidden. The bigrams and finishing word should in no way be related to the message.

Padding and group separation

The plaintext is divided into groups of four letters. If the last group contains fewer than four letters, it is padded with any consonants. Letters should be chosen as to avoid confusion.

Padding must not be chosen from text aids or other common letter combinations as this would make cryptanalysis easier.

Messages should be encrypted in 250 characters or less. Longer messages are split up and each part encrypted with a different key

Splitting of messages

If a message needs to be divided into several messages, the word FORT (fortsetzung, continuation) is inserted somewhere in the last third of every message except the last. In each message except the first, FORT is inserted in the first third, in the same manner, followed by the time group of the preceding message.

The last message should make special note of the time group of the first message.

Each message should be encrypted with a different key.
